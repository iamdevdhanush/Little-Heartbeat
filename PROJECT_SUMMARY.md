## SECTION 1 — EXECUTIVE SUMMARY

**Project Name:** Little Heartbeat

**One-line Description:** The Pregnancy Operating System — a single application that manages a woman's entire pregnancy journey through AI-powered document intelligence, medication tracking, timeline management, and emergency safety.

**Elevator Pitch:** Every pregnant woman today juggles 4-6 apps for baby development, symptom checking, medication reminders, doctor appointments, and health records. Little Heartbeat replaces them all with one unified Pregnancy Operating System. Upload a prescription, and AI automatically extracts medicines, dosages, timing, and appointments. It's the Apple Health + Flo + Oura experience — purpose-built for pregnancy.

**Vision Statement:** To be the central nervous system of every pregnancy — the first screen a pregnant woman opens every day, and the one app she trusts through all nine months.

**Mission Statement:** Empower every pregnant woman with intelligent, compassionate technology that makes her feel safe, supported, and in control throughout her pregnancy journey — replacing confusion with clarity, fragmentation with unity, and anxiety with confidence.

**Why It Exists:** Pregnancy apps today are either symptom checkers, content libraries, or generic trackers. None serve as a comprehensive operating system. Women piece together multiple tools, creating friction, missed medications, disconnected health records, and unnecessary stress. Little Heartbeat exists because pregnancy deserves better than duct-taped healthcare apps.

**Why It Matters:** 130 million babies are born every year. Every pregnancy is high-stakes. Fragmented information leads to missed doses, forgotten appointments, lost reports, and delayed emergency response. An integrated pregnancy OS is not convenience — it's a safety necessity.

## SECTION 3 — PRODUCT OVERVIEW

### Core Concept

Little Heartbeat is not an app. It is a Pregnancy Operating System — the central, unified platform where a pregnant woman manages her entire pregnancy journey from conception to postpartum.

### Product Philosophy

1. **Operating System, Not An App** — Apps are individual tools. An OS is the environment where everything lives. We don't add features; we provide the infrastructure for pregnancy management.

2. **AI-Assisted, Not AI-Driven** — AI extracts, summarizes, reminds, and suggests. It never diagnoses. The doctor remains the authority. AI is a tireless assistant, not a replacement.

3. **Emotional First, Clinical Second** — Every interaction starts with emotional safety. Clinical information is presented within a warm, supportive context. The user feels cared for, not monitored.

4. **Zero Friction, Maximum Coverage** — Upload a prescription → automatically extracted, scheduled, and stored. No manual entry. No data loss. Complete coverage.

### User Experience

```
Day 1: Download → Onboard → Upload prescription → AI extracts medicines
       → Set reminders → View baby size → Feel supported

Daily:  Open → See greeting → Check medication → Log mood/symptoms
        → View baby development → Check appointment → Done (45 seconds)

Weekly: Timeline check → New insights → Upload new reports
        → Review adherence → Partner sync

Monthly: Full health review → Trend analysis → Report organization
         → Doctor prep
```

### Product Pillars

1. **Medical Document Intelligence** — Upload any medical document. AI extracts everything meaningful: medicines, dosages, timing, appointments, tests, instructions.

2. **Medication Intelligence** — Automatic reminders, adherence tracking, habit learning, compliance monitoring, timeline visualization.

3. **Pregnancy Timeline** — Week-by-week journey with baby development, milestones, appointments, reports, and health events in one scrollable timeline.

4. **Health Vault** — Secure document storage with AI summaries, intelligent search, medical history, and trend tracking.

5. **Emergency Safety** — One-tap SOS with GPS sharing, emergency contacts, medical ID card, hospital recommendations.

## SECTION 5 — CORE FEATURES

### 1. Today Dashboard

**Purpose:** The daily home screen — the first thing users see. Provides an emotional greeting, baby development update, today's tasks, and quick access to key actions.

**User Value:** 45-second daily check-in. Knows baby size, medication status, mood, appointment. Feels supported and guided.

**Business Value:** Primary engagement surface. Drives daily retention. Showcases AI capabilities.

**Technical Complexity:** Low-Medium. Interactive components (mood toggles, symptom chips, water tracker, medication check). 3 upload states (idle/uploading/success).

**Usage Frequency:** Daily (multiple times)

### 2. Pregnancy Timeline

**Purpose:** Week-by-week journey visualization. Past milestones, current week hero, upcoming events.

**User Value:** Sees the entire pregnancy at a glance. Celebrates milestones. Prepares for upcoming events.

**Business Value:** High emotional engagement. Differentiator from generic calendars.

**Technical Complexity:** Medium. Timeline rendering, milestone status (past/current/future), animated transitions, trimester filtering.

**Usage Frequency:** Weekly

### 3. Health Vault

**Purpose:** Secure document storage with AI-powered summaries, search, and organization.

**User Value:** All medical documents in one place. AI extracts key values. Searchable. Never lose a report again.

**Business Value:** High retention (stored data creates switching cost). Showcases AI capabilities. Document Intelligence core differentiator.

**Technical Complexity:** High. Document storage, AI extraction pipeline, search indexing, category management, file upload with progress.

**Usage Frequency:** Weekly (upload) / Monthly (review)

### 4. Medication Center (Care)

**Purpose:** Complete medication management with adherence tracking, schedule, and refill management.

**User Value:** Never miss a dose. See adherence rates. Track all medications in one place.

**Business Value:** High stickiness. Daily engagement driver. Compliance data valuable for healthcare partnerships.

**Technical Complexity:** Medium-High. Schedule management, push notifications, adherence calculation, refill tracking, toggle interactions.

**Usage Frequency:** Multiple times daily

### 5. Emergency SOS

**Purpose:** One-tap emergency alert with GPS location sharing, contact notification, and medical ID card.

**User Value:** Critical safety net. Quick access to emergency services. Contacts automatically notified with location.

**Business Value:** Trust builder. High emotional value. Critical differentiator.

**Technical Complexity:** Medium-High. GPS integration, WhatsApp sharing, SMS gateway, medical ID display, countdown timer.

**Usage Frequency:** Rare (but critical when needed)

### 6. Health Insights (AI)

**Purpose:** Personalized daily insights about baby development, body changes, and health tips.

**User Value:** Understands what's happening with her body and baby. Feels guided and informed.

**Business Value:** Daily engagement driver. AI showcase. Personalization value.

**Technical Complexity:** Medium. AI-generated content, personalization, weekly/monthly adaptation.

**Usage Frequency:** Daily

### 7. Appointment Tracking

**Purpose:** Doctor appointment management with reminders, details, and preparation.

**User Value:** Never miss an appointment. Has all details ready. Can reschedule easily.

**Business Value:** Engagement driver. Data collection for personalization.

**Technical Complexity:** Low-Medium. Calendar integration, reminders, doctor information.

### Architecture Decisions

**Why PWA + React:** Cross-platform from day one. No iOS/Android fragmentation. Instant updates without app store review. Offline support for critical features (SOS, medication data). Lower development cost.

**Why Supabase:** Provides authentication, database, storage, and real-time in one platform. Scales from prototype to millions of users. Row-level security for HIPAA-style data protection.

**Why Google Gemini AI:** State-of-the-art document understanding. Can extract structured data from unstructured medical documents. Available API. Fast inference.

**Why Local-First Architecture:** Critical medical data must be accessible offline. PWA Service Worker caches assets and data. LocalStorage for profile and preferences. Supabase synced when online.

## SECTION 9 — PWA STRATEGY

### Why PWA for Pregnancy

1. **Zero Installation Friction** — Search → Open → Use. No app store download. Critical when a woman discovers she's pregnant and needs information immediately.

2. **Offline-First Medical Access** — Medication schedules, emergency contacts, and medical ID must work without internet. PWA Service Worker enables this.

3. **Always Up-to-Date** — Medical information changes. PWA updates automatically. No "update available" prompts. No app store review delays.

4. **Cross-Platform Unity** — iOS and Android share the same codebase. No feature disparity. No delayed Android releases. No iOS review rejections.

5. **Discoverable by Search** — Content indexed by Google. "Pregnancy symptom" searches can lead directly to the app. Native apps can't do this.

6. **Lower Development Cost** — One codebase. ~60% cost savings vs. native. Faster iteration. Smaller team required.

### PWA Architecture

| Layer | Technology | Strategy |
|---|---|---|
| App Shell | Vite + React | Precache via Workbox |
| Static Assets | Workbox precaching | Cache-first, update on revision |
| API Data | Workbox runtime caching | Network-first with cache fallback |
| User Data | LocalStorage + IndexedDB | Local-first, sync when online |
| Offline SOS | Service Worker | Queue SOS request, send when online |
| Push Notifications | Push API + Notification API | Reminders for medications, appointments |
| Install Prompt | beforeinstallprompt event | Custom frosted glass prompt |
| Splash Screen | Manifest + theme-color | Branded loading experience |
| Shortcuts | Manifest shortcuts | Direct SOS, Health Check, Vault access |

## SECTION 10 — AI SYSTEM

### Architecture

```
User Uploads Document
        │
        ▼
┌─────────────────────┐
│  Document Preprocess │  ← Resize, format conversion, OCR prep
│  (Client-side)       │
└─────────────────────┘
        │
        ▼
┌─────────────────────┐
│  Gemini AI API       │  ← Structured extraction prompt
│  Extract Medicines   │
│  • Name & Dosage     │
│  • Timing & Frequency│
│  • Doctor Info       │
│  • Appointment Dates │
│  • Lab Test Values   │
└─────────────────────┘
        │
        ▼
┌─────────────────────┐
│  Post-Process        │  ← Validate, format, categorize
│  • Structure data    │
│  • Calculate dates   │
│  • Create reminders  │
│  • Update timeline   │
└─────────────────────┘
        │
        ▼
┌─────────────────────┐
│  User Review         │  ← User confirms or corrects
│  • Confirm medicines │
│  • Adjust timings    │
│  • Approve reminders │
└─────────────────────┘
```

### AI Capabilities

| Capability | Input | Output | Accuracy Target |
|---|---|---|---|
| Prescription Extraction | Photo/PDF | Medicines, dosages, timing, frequency | > 95% |
| Report Summarization | Lab report | Key values, flags, trends, plain English summary | > 90% |
| Medication Schedule | Prescription data | Calendar of doses, adherence tracking | > 98% |
| Health Insights | User profile + week | Daily personalized guidance | Contextual |
| Symptom Analysis | User input | Guidance, when to see doctor | Informational |
| Appointment Extraction | Doctor note | Date, time, location, doctor, purpose | > 90% |

### Safeguards

1. **AI Assists, Never Diagnoses** — Every AI output includes: "This is not a medical diagnosis. Always consult your doctor."

2. **User Confirmation Required** — All extracted data is presented for user review before being saved.

3. **Disclaimer Prominently Displayed** — App-wide disclaimer in multiple locations.

4. **No Treatment Recommendations** — AI does not suggest treatments, change dosages, or override doctor instructions.

5. **Emergency Keywords Detected** — If extraction or user input indicates potential emergency, SOS system is suggested, not triggered.

6. **Data Privacy** — Documents processed through Gemini API with no training on user data. Supabase RLS ensures user isolation.

## SECTION 16 — SECURITY & PRIVACY

### Data Protection Strategy

| Data Type | Storage | Encryption | Access |
|---|---|---|---|
| User Profile | Supabase DB | At-rest + in-transit | User only (RLS) |
| Medical Documents | Supabase Storage | Server-side encryption | User only |
| AI Analysis Results | Supabase DB | At-rest + in-transit | User only |
| Medication Data | Local + Supabase | Local: IndexedDB encrypted | User only |
| Emergency Contacts | Supabase DB | At-rest encrypted | User + emergency flow |
| GPS Location | In-memory + temporary | Not stored long-term | Shared only during SOS |
| Mood/Symptom Logs | Supabase DB | At-rest encrypted | User only |

### Authentication

- Supabase Auth with email/password and OAuth providers
- Row-Level Security (RLS) on all database tables
- Policies ensure users can only access their own data
- Session management with automatic refresh

### Medical Data Compliance

While not HIPAA-certified at V1, the architecture is designed for HIPAA compliance:
- Row-level security for complete data isolation
- Encryption at rest and in transit
- Audit logging capability
- Access control policies
- Data portability (export all data)
- Data deletion capability

### Privacy Strategy

1. **Minimal Data Collection** — Only data necessary for core function
2. **No Data Selling** — User data is never sold or shared with third parties
3. **Transparent AI** — Users know when AI is processing their data
4. **User Control** — Full access to download or delete all data
5. **Anonymized Analytics** — Only anonymous usage patterns tracked
6. **GDPR Ready** — Architecture supports right to access, deletion, portability
