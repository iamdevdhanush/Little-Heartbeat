## Frontend Architecture

```
main.jsx
  └── AppProvider (context/AppContext.js)
        └── ReminderProvider (reminder/ReminderProvider.jsx)
              └── App (App.jsx)
                    └── BrowserRouter
                          ├── AppLayout (layout/shell + bottom nav)
                          └── 8 Route Pages
                                ├── /          → HomePage (dashboard)
                                ├── /baby      → BabyPage (growth viz)
                                ├── /ai        → AIPage (chat)
                                ├── /care      → CarePage (medications)
                                ├── /journey   → JourneyPage (timeline)
                                ├── /sos       → SOSPage (emergency)
                                ├── /profile   → ProfilePage (settings)
                                └── /upload    → UploadPage (prescription)
```

### Data Flow
- Pages → Hooks → Services → Supabase (direct) + localStorage (fallback)
- AI Chat → aiService.js → Gemini API (direct, API key in env var)
- OCR → ai/extraction/ → tesseract.js + pdfjs-dist (browser-side)
- Reminders → reminder/ → Notification API + Service Worker

## Backend Architecture

```
main.py → lifespan (auto-create tables) → CORS → 7 routers
  ├── /api/auth/          → JWT login + user profile
  ├── /api/prescriptions/ → Upload + OCR + confirm
  ├── /api/medications/   → List + confirm take/skip
  ├── /api/appointments/  → CRUD
  ├── /api/timeline/      → List by week
  ├── /api/emergency/     → CRUD contacts
  ├── /api/uploads/       → Document upload
  └── /api/health         → DB connectivity check
```

### Known Divergence
The frontend currently communicates directly with Supabase and Gemini,
bypassing the local FastAPI backend. The backend is wired and functional
but not consumed by the web frontend. This is preserved as-is — the
backend serves as the API layer for future mobile clients.

## Key Design Decisions
- Vite 6 + React 18 (not Expo/Next.js)
- Direct Supabase queries for CRUD (backend API exists but unused by web)
- Browser-side OCR (tesseract.js + pdfjs-dist) — backend also has Python OCR
- Zustand for state management (reminder store)
- Context API for app-level state (profile, language)
- localStorage as offline fallback (dual Supabase + local data path)
